# Agent context — docker-mise

Umbrella repository for mise base development images. Flavors live as **git submodules** on GitHub.

**Default remote name:** `github` (not `origin`)  
**Default branch:** `master`  
**Last updated:** 2026-07-16

## Layout

```
docker-mise/
├── AGENTS.md
├── README.md
├── ubuntu-mise/   # submodule → Ruby-on-Rails-Wizardry/ubuntu-mise
├── alpine-mise/   # submodule → Ruby-on-Rails-Wizardry/alpine-mise
└── arch-mise/     # submodule → Ruby-on-Rails-Wizardry/arch-mise
```

## Submodules

| Path | GitHub URL |
|------|------------|
| `ubuntu-mise` | `git@github.com:Ruby-on-Rails-Wizardry/ubuntu-mise.git` |
| `alpine-mise` | `git@github.com:Ruby-on-Rails-Wizardry/alpine-mise.git` |
| `arch-mise` | `git@github.com:Ruby-on-Rails-Wizardry/arch-mise.git` |

Clone: `git clone --recurse-submodules -b master git@github.com:Ruby-on-Rails-Wizardry/docker-mise.git`

Update pins: `git submodule update --remote --merge` (then commit parent).

## Per-flavor remotes

Inside each submodule, **github** is default and **gitlab** is backup (`./bin/setup-remotes`).

## Host UX (inside a flavor)

| Command | Purpose |
|---------|---------|
| `task setup` / `./bin/setup` | Build image + cache volume + warm |
| `task shell` / `./bin/shell` | docker run path |
| `task compose:shell` / `./bin/compose-shell` | Compose path |

Do not commit secrets or `.env` files generated by `bin/compose`.
